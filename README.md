# Интерактивный Календарь Встреч Компании

Современный и премиальный веб-интерфейс для планирования встреч сотрудников компании с автоматической проверкой пересечений времени (Double-booking prevention) и возможностью комментирования встреч в реальном времени.

Проект разработан в рамках тестового задания для ИИ-специалиста.

---

## 🚀 Возможности проекта

1. **Планирование встреч**: Добавление темы, даты, времени начала/окончания, участников и повестки/описания встречи.
2. **Просмотр расписания**:
   - **Вид "Месяц"**: Классическая сетка дней с цветными карточками встреч и быстрой возможностью добавить встречу на любой день в один клик.
   - **Вид "Неделя"**: Распределение встреч по колонкам дней недели (Пн-Вс).
   - **Вид "Повестка дня"**: Хронологический список всех предстоящих встреч на ближайшие 30 дней.
3. **Предотвращение пересечений (Conflict Validation)**: 
   - Бэкенд автоматически проверяет, не занят ли кто-то из указанных участников в других встречах на выбранный интервал времени.
   - Если обнаруживается совпадение, интерфейс выводит подробное Toast-уведомление с перечислением конфликтующих сотрудников, а также тем и времени встреч, в которых они задействованы.
4. **Обсуждение встреч**: Любой сотрудник может открыть детали встречи, ознакомиться со списком участников и повесткой, а также написать комментарий в живом чате обсуждения.
5. **Динамическая настройка API**: Статический фронтенд имеет выпадающую панель настроек, где можно указать URL бэкенда (например, публичный IP вашего VPS). Это позволяет хостить фронтенд на GitHub Pages, а бэкенд на VPS/Serverbook без изменения кода.
6. **Выбор темы оформления**: В сайдбар встроен переключатель тем в 4 стилях:
   - **Тёмная космическая** (фирменный стиль Antigravity с неоновым свечением).
   - **Светлая классическая** (аккуратный деловой интерфейс с мягкими тенями).
   - **Стиль Wildberries** (фиолетово-малиновые тона маркетплейса).
   - **Стиль Ozon** (фирменный ярко-синий и розовый цвета).

## 🛠️ Стек технологий

*   **Фронтенд**: HTML5, CSS3 (Vanilla с кастомными переменными, CSS Grid, Flexbox, стеклянным эффектом glassmorphism), Vanilla JavaScript, FontAwesome.
*   **Бэкенд**: Python 3.11, FastAPI, Uvicorn, SQLite3, Pydantic (валидация данных).
*   **Контейнеризация**: Docker, Docker Compose.

## ⚙️ Запуск и Развёртывание

### 1. Локальный запуск бэкенда (Python)

Убедитесь, что у вас установлен Python 3.11+.

```bash
# Перейдите в папку бэкенда
cd calendar/backend

# Создайте и активируйте виртуальное окружение
python3 -m venv venv
source venv/bin/activate  # Для Linux/macOS
# venv\Scripts\activate   # Для Windows

# Установите зависимости
pip install -r requirements.txt

# Запустите сервер разработки
uvicorn app.main:app --reload --port 8000
```

Сервер запустится по адресу `http://localhost:8000`.
*   Swagger-документация API доступна по адресу: `http://localhost:8000/docs`.
*   Файл базы данных `meetings.db` создастся автоматически при первом запуске в папке `calendar/backend`.

---

### 2. Запуск бэкенда через Docker / Docker Compose (на VPS / Serverbook)

Для быстрого развёртывания бэкенда на сервере выполните:

```bash
# Перейдите в корневую папку проекта (где лежит docker-compose.yml)
cd calendar

# Соберите и запустите контейнер в фоновом режиме
docker compose up -d --build
```

Контейнер автоматически инициализирует базу данных во внутреннем томе Docker `db-data` и откроет порт `8000` для входящих запросов к API.

---

### 3. Запуск фронтенда

Поскольку фронтенд написан на чистом стеке HTML/CSS/JS без фреймворков и сборщиков:
1. Вы можете просто открыть файл `calendar/frontend/index.html` двойным кликом мыши в любом браузере.
2. Или запустить любой легкий статический веб-сервер, например:
   ```bash
   cd calendar/frontend
   python3 -m http.server 3000
   ```
   После чего открыть в браузере `http://localhost:3000`.

---

### 4. Хостинг фронтенда на GitHub Pages

Вы можете загрузить папку `frontend` в отдельный репозиторий GitHub или в качестве подпапки и включить GitHub Pages в настройках репозитория.

**Настройка связи с бэкендом:**
1. Откройте опубликованный сайт на GitHub Pages.
2. В левом нижнем углу сайдбара нажмите на **Настройки API**.
3. Введите адрес вашего запущенного бэкенда (например, `http://95.214.8.10:8000` или домен вашего VPS).
4. Нажмите кнопку **Применить**.
5. Статус API сменится на зеленый маркер "Онлайн", и календарь начнет загружать данные с вашего сервера.

## 🔍 Алгоритм валидации конфликтов (Double-booking)

В файле `calendar/backend/app/main.py` при создании встречи выполняется следующая логика:
1. Выполняется SQL-запрос к SQLite для поиска встреч, пересекающихся с целевым интервалом времени `[StartNew, EndNew]`:
   ```sql
   SELECT * FROM meetings WHERE start_time < EndNew AND end_time > StartNew
   ```
2. Для каждого найденного пересечения парсится JSON-список участников (`participants`).
3. Если хотя бы одно имя из участников новой встречи (без учета регистра) совпадает со списком занятых участников, формируется детальный список конфликтов.
4. Сервер возвращает ошибку `409 Conflict` с JSON-структурой, описывающей, кто именно занят, в какой встрече и в какое время.
5. Фронтенд перехватывает этот ответ и выводит красивое всплывающее окно с предупреждением.
